- **「初の有料意図」は 06-12 ではない。** 事業として最も収益に近づいたのは **2026-05-26 の実トライアル**（後述・ポーランド企業）。

- IP `106.73.230.226` は **日本・栃木・日光（proxy:true）**。experience-villa のある那須塩原に隣接、かつこのIPは **04-27 に experience-villa.com（自社サイト）をスキャン済み**。

### 7.5 経路コードの確認（ボタンは壊れていない）

- `src/app/api/checkout/route.ts`: `stripe.checkout.sessions.create()` **成功後**に server 側 `checkout_clicked` を記録（L76→L106）。06-12 にイベントが出た＝**Stripe チェックアウトセッションは正常に生成され URL が返っている**。ボタン/経路の不具合ではない。
- トライアルは `payment_method_collection: "if_required"`（カード不要トライアル）。開始摩擦は最小だが、**カード未登録のため7日後に静かに失効**する設計。05-26→06-02 の churn はこの通りの挙動。

### 7.6 真の課題：トライアル・ライフサイクルのメールが皆無

- `email_events` の全期間 kind は **`scan_report` のみ（5件）**。`welcome` / `trial_started` / `trial_ending` / `trial_ended` / `payment_failed` / `win_back` は**一つも存在しない**。
- ポーランド prospect が受け取ったメールは 05-22 の scan_report 1通のみ。トライアル中・終了前・失効後の接触ゼロ。
- webhook（`src/app/api/webhook/route.ts`）は trial_started / canceled を**記録するがメールは送っていない**。
- **→ トライアルが無通知で失効し転換圧がかからない**のが、0転換の構造的原因。ここを仕組みで埋めるのが endgame で最も効く一手。
